About Me

Hello, I am Sam — an IT professional with a strong foundation in Computer Information Systems and hands-on experience supporting enterprise environments.

I currently provide technical support across a range of retail environments, handling both end-user issues and on-site technician tasks. My responsibilities include troubleshooting hardware and software problems, performing device installations and repairs, and organizing cable infrastructure to support reliable network performance. In addition to help desk support, I contribute to small-scale development and configuration projects, allowing me to deliver practical, user-focused solutions that align with organizational needs.

I hold an Associate of Applied Science in Computer Information Systems from Davenport University and have completed a range of personal and academic projects involving web development, mobile apps, and system administration. I’m comfortable working independently, adapting to evolving systems, and delivering responsive support across a wide range of technologies—from Active Directory and Microsoft 365 products to custom-built applications.

Whether I’m designing a front-end interface, automating workflows, or configuring servers in a virtual lab, I’m driven by curiosity, problem-solving, and the pursuit of elegant, functional solutions.

Thank you for visiting—I invite you to explore my portfolio and reach out if you’d like to connect.

Employee-Identification-Process

January 2025

Repository Temporarily Unavailable Untill 4/26

This project implements a multi-factor authentication (MFA) workflow using the DUO Security SDK within an ASP.NET C# application. Designed for employee verification, the system leverages DUO push notifications as a secondary authentication layer. After initial login with standard credentials, users are prompted with a DUO push challenge before gaining access to a secure dashboard. Within this protected area, administrators can issue MFA challenges to verify employee identities. While fully functional with valid API keys, the project surfaced concerns around potential social engineering vulnerabilities and limitations in DUO’s use case, which ultimately led to its suspension.

  • Built an MFA authentication portal using ASP.NET (C#) and the DUO Security SDK
  • Designed a dual-layer login process involving standard credentials followed by DUO push verification
  • Implemented a protected dashboard using ASP.NET authentication mechanisms
  • Enabled administrators to issue DUO push challenges for employee identity verification
  • Addressed integration with DUO’s API for real-time authentication workflows
  • Identified and evaluated social engineering risks related to false MFA prompts
  • Project remains functional with valid DUO API keys but was suspended due to security and UX limitations

Home Lab

July 2024

This project was a self-built Windows lab environment designed as a hands-on playground to explore and understand domain-based networking and basic IT support tasks. It involved setting up a Windows Server 2022 Domain Controller with Active Directory, DNS, DHCP, and Group Policy, then joining Windows 10 virtual machines to the domain. The lab simulated real-world IT help desk responsibilities such as domaining PCs, creating users and managing passwords, setting folder permissions, configuring shared drives, and applying GPOs. It provided valuable experience in troubleshooting and managing a small domain network from the ground up.

  • Deployed a virtualized home lab environment using Oracle VM VirtualBox.
  • Installed and configured Windows Server 2022 as a Domain Controller (DC).
  • Set up and managed Active Directory (AD) to create and organize user accounts (General, HR, IT).
  • Configured DNS and DHCP services on the Domain Controller to handle name resolution and IP assignment.
  • Joined two Windows 10 virtual machines to the domain, simulating a small enterprise network.
  • Designed and applied Group Policy Objects (GPOs) for security, user restrictions, and desktop configurations.
  • Created and managed Network Shared Drives, with permissions based on AD group memberships.
  • Set up Remote Desktop Protocol (RDP) for remote management and user access across virtual machines.
  • Demonstrated knowledge of user/group management, domain networking, remote access, and Windows Server administration.
